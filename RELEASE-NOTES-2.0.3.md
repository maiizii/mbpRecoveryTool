# RELEASE NOTES - v2.0.3

发布日期：2026-03-27

## 本次更新

这是基于 v2.0.2 的小版本更新，重点是把“用户检索”和“恢复任务”彻底解耦，并补齐 NAS MBP 包恢复能力。

### 新增 / 调整

- 恢复任务改为手动输入 UID，不再依赖“用户检索”结果作为前置条件。
- 恢复任务新增“文件位置”选择：
  - `Box` → `/mmc/mbp/<uid>.mbp`
  - `NAS` → `/mnt/myt/mbp/<uid>.mbp`
- 用户检索与恢复任务配置拆分：
  - 检索走 `detect`
  - 恢复走 `recover`
- 前端连接选择逻辑解耦，避免检索页/恢复页互相串写配置。

### 效果

- 用户 MBP 包可以从 NAS 独立取用，不再强绑定某个盒子或机位。
- 恢复链路更直接，适合后续逐步把用户 MBP 包统一迁到 NAS。

### 说明

- 基座文件仍建议保留在盒子本地，以避免大文件经 NAS 复制带来的速度与稳定性问题。
- 当前恢复主流程结束时会清理盒子侧临时工作目录 `/mmc/myt_recover_work/<uid>`；异常中断场景下仍可能残留，下次任务可覆盖/清理。
