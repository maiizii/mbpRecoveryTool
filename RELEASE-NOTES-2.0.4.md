# RELEASE NOTES - v2.0.4

发布日期：2026-03-27

## 本次更新

这是基于 v2.0.3 的小版本更新，重点修复“恢复任务”在 S5 阶段可能错误复用旧检测缓存代理 IP 的问题，并增加一个可配置的 socks5 回退代理。

### 修复

- S5 阶段不再读取 `config.recover.detected.proxyIp` 的缓存值；改为在恢复主流程中，从**当前用户** MBP 解包目录中解析外网 IP（优先从 `location.json` / `baseCfg.json` 等文件树中提取）。
- 当代理映射文件按外网 IP 反查失败时，不再直接失败：支持回退到“通用 socks5 代理”。

### 新增 / 调整

- 其他设置页新增：**通用 socks5 代理** 输入框
  - 格式：`网关:端口:用户名:密码`
  - 用途：当外网 IP 在代理映射表中找不到时，回退使用该代理进行 S5 设置。
- 后端配置新增：`recover.fallbackSocks5`

### 说明

- 代理映射命中时仍优先使用映射表结果；回退仅在映射 miss 时触发。
- 新的外网 IP 解析逻辑依赖 MBP 解包目录存在（恢复流程本身会先准备并解包 MBP）。
